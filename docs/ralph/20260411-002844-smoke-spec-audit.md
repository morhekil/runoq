# Smoke Spec Audit

Date: 2026-04-11 00:28:44 +1100

## Conclusion

The draft smoke spec is largely aligned with the current implemented runtime's externally observable behavior. I found one concrete runtime defect that causes divergence from the documented behavior.

## Findings

1. System bug: pending planning review without a proposal can be misclassified as awaiting human decision.

The spec says a pending planning review with no proposal payload in the issue body should be redispatched through the normal planning-dispatch path.

Current implementation in `internal/orchestrator/tick.go` checks for `runoq:payload:plan-proposal` by searching the entire JSON returned from `gh issue view --json number,title,body,comments,labels,state`. Because that payload includes comments as well as the body, a comment containing the marker can make the tick believe a proposal exists even when the issue body does not contain one.

Externally observable impact:

- `runoq tick` can print `Awaiting human decision on #<n>` instead of redispatching planning.
- The system can stall a planning issue that should still be in the decomposition lane.

Classification:

- Spec: correct.
- System: incorrect; requires a code fix and regression coverage.

Resolution:

- Fixed in runtime by making pending-review proposal detection inspect the review issue body only.
- Added regression coverage in `internal/orchestrator/tick_test.go`.

## Missing Scenarios

No additional high-signal happy-path or failure-path gaps were identified beyond the existing spec coverage during this audit pass.
